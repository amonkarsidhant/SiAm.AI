# Onboarding

The Onboarding project establishes the operating system of SiAm.AI.

## Purpose

Build the company foundation so the executive team can operate with clarity, rhythm, and useful delegation.

## Outcomes

This project should produce:
- clear executive role boundaries
- an operating cadence for reviews and delegation
- initial company planning artifacts
- hiring clarity for the next critical role
- the first usable project and task structure inside Paperclip

## Scope

Included:
- company charter and thesis
- executive role definitions
- delegation and planning standards
- first hiring decisions
- first-wave task creation
- operating rhythm and review loop

Excluded:
- full product build execution
- broad long-term hiring sprawl
- deep brand work before market focus
- large-scale automation before process clarity

## Success Criteria

The Onboarding project is successful when:
- CEO, CTO, and CMO each have actionable mandates
- the company has a clear first priority order
- the first B2B exploration path is explicit
- the next critical hire is identified or rejected with reason
- the board can inspect the company and understand what happens next

## Priority Order

1. company operating clarity
2. first B2B exploration framework
3. first role and hiring decisions
4. first validation path

## Risks

- too much strategy without delegation
- too many parallel priorities
- vague task definitions
- hiring before role clarity
- B2C distracting from stronger B2B opportunities

## Project Standard

This project should bias toward practical outputs, not essays.
